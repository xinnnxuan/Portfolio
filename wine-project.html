Wine Rating Classification Project

CPSC 322 - Fall 2024 Team: Johanna Fan & Lindsey Bodenbender GitHub Repository Dataset Source

Project Overview

Wine is a complex product influenced by price, vintage, winery reputation, and consumer perception. This project applies machine learning algorithms to classify and predict wine ratings based on structured data. Using a combination of decision trees, Naïve Bayes, and random forests, we explore which factors best determine wine quality.

Objectives

  • Predict wine ratings (Poor, OK, Average, Good, Excellent) based on attributes
  • Analyze relationships between price, year, number of ratings, and wine quality
  • Build a Flask-based web app for interactive predictions
  • Identify challenges in rating distributions and data bias

Exploratory Data Analysis

  • Price Distribution → Right-skewed, most wines are lower-priced
  • Year of Production → Most wines were produced after 2016
  • Number of Ratings → A few wines dominate in reviews, leading to imbalance
  • Correlation Analysis → Higher-priced wines tend to have better ratings

Machine Learning Implementation

Classification Models

  • Naïve Bayes → Probabilistic model assuming independence among features
  • Decision Tree → Splits data into decision-based paths
  • Random Forest → Uses multiple decision trees for higher accuracy

Training & Testing

  • Cross-Validation: k=10 Random Subsampling
  • Features Used: Price, Year, Number of Ratings

Model Performance

Model Accuracy

Web App & API

How It Works

  • Users enter Price, Year, and Number of Ratings
  • The Decision Tree Model predicts a rating category
  • Output: "Poor", "OK", "Average", "Good", "Excellent"

Tech Stack

  • Backend: Flask API to handle requests
  • Model Storage: Trained classifiers are saved using Pickle
  • Frontend: A user-friendly interface to input wine details

Key Findings & Insights

Data Skewness

Price and number of ratings are heavily right-skewed

Class Imbalance

A small number of wines have extremely high reviews

Feature Selection

Adding more attributes could improve predictions

Performance Insights

  • Decision Trees outperformed Naïve Bayes and Random Forests
  • Price had a stronger impact on rating predictions than vintage year
  • Handling missing values played a crucial role in model accuracy

Future Enhancements

  • Enhance Feature Engineering → Include factors like grape variety, winery reputation, and critic reviews
  • Improve Model Performance → Optimize classifiers with hyperparameter tuning
  • Deep Learning Approach → Train a Neural Network for higher predictive accuracy
  • Expand Web App → Add visualizations and user interaction features
